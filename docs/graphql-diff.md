## 6\. 분석: 요청별 주요 차이점

### 상세 분석

#### 1\. API 엔드포인트 (`OperationName`) 분리

검색 대상의 주요 카테고리별로 명확하게 구분된 전용 API를 호출합니다.

  - **병원**: `getNxList` (의료 정보 전용)
  - **미용실**: `getBeautyList` (뷰티/미용 정보 전용)
  - **맛집**: `getRestaurants` (음식점 정보 전용)
  - **숙박**: `getAccommodationList` (숙박 정보 전용)
  - **일반 장소**: `getPlacesList` (범용 장소 정보)

#### 2\. 카테고리별 특화 데이터 요청

각 API는 해당 카테고리의 특성에 맞는 고유한 데이터 필드를 요청하여 사용자에게 더 유용한 정보를 제공합니다.

  - **병원 검색 시**: 건강보험심사평가원 전문의 정보(`hiraSpecialists`), 의료기관 번호(`medicalNo`) 등 전문적인 의료 정보를 요청합니다.
  - **미용실 검색 시**: 대표 시술 가격(`representativePrice`), 스타일 개수(`stylesCount`), 스타일 이미지(`styleImages`) 등 미용실에 특화된 정보를 요청합니다.
  - **맛집 검색 시**: 미쉐린 가이드 정보(`michelinGuide`), 인기 메뉴(`popularMenuImages`), 방송 출연 이력(`broadcastInfo`) 등 음식점에 특화된 상세 정보를 요청합니다.
  - **숙박 검색 시**: 객실 이미지(`roomImages`), 평균 가격(`avgPrice`), 편의시설(`facility`), 예약자 수(`bookingUserCount`) 등 숙박 시설에 특화된 정보를 요청합니다.
  - **일반 장소 검색 시**: 모든 장소 유형에 적용될 수 있는 범용적인 정보(주소, 전화번호, 영업시간 등)를 기본으로 요청합니다.

#### 3\. 광고 요청의 명확한 구분

모든 경우에 `getAdBusinessList` API를 통해 광고 데이터를 별도로 요청합니다. 이때 `variables.input.businessType` 값을 `hospital`, `hairshop`, `restaurant`, `accommodation`, `place`로 각각 다르게 설정하여, 검색어의 종류에 맞는 광고를 정확하게 받아오도록 설계되어 있습니다.
